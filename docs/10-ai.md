# 10 — AI: BFS Pathfinding

> **In one sentence:** Each tick, the AI runs a **Breadth-First Search** from the snake's head to the food, treating the snake's body as obstacles, and takes the first step along the shortest path.

---

## What is BFS?

**Breadth-First Search** is the textbook algorithm for finding the shortest path on a grid (or any unweighted graph).

The intuition:
1. Stand at the start.
2. Mark every neighbor as "1 step away".
3. From each of those, mark every new neighbor as "2 steps away".
4. Keep expanding outward in layers.
5. Stop when you reach the target — you've found the shortest path.

It's like dropping a stone in water and watching the ripples spread.

### Diagram (small board, target = `F`)

Iteration order from snake head `H`:

```
  . . . . .                         3 . . . .
  . . . F .                         2 1 . . .
  . . H . .         after BFS →     1 H 1 . .         (numbers = distance)
  . . . . .                         2 1 2 . .
  . . . . .                         3 . . . .
```

We keep going until the cell containing `F` gets a number — that's the shortest distance. Then we **walk back** from `F` to `H` via the recorded "where did this number come from" pointers, and the **first step** of that walk is the move we should make.

---

## Why BFS for snake?

- The grid is **unweighted** (every move is one step) → BFS gives the optimal path.
- It handles **obstacles** naturally — we just don't enqueue cells that have a snake body.
- It supports **wraparound** as long as we wrap the neighbor coordinates.
- It's fast enough to run **every tick** on a 200×60 board (12,000 cells).

---

## Data structures

We use **two static 2D arrays** for `visited` and `parent` (direction we came from), plus a queue:

```c
#define AI_MAX_W 210
#define AI_MAX_H 70
#define AI_MAX_CELLS (AI_MAX_W * AI_MAX_H)

static int  visited[AI_MAX_H][AI_MAX_W];
static int  parent[AI_MAX_H][AI_MAX_W];   // direction (0..3) to enter this cell
static int  qx[AI_MAX_CELLS];             // queue X
static int  qy[AI_MAX_CELLS];             // queue Y
```

A few things to notice:
- **Static arrays** (file-scope) — never freed, sized to the maximum board.
- The "queue" is just two parallel arrays with `qhead` and `qtail` indices — no dynamic allocation needed.

### Direction encoding

```c
static const int DX[4] = { 0,  0, -1,  1 };   // UP, DOWN, LEFT, RIGHT
static const int DY[4] = {-1,  1,  0,  0 };
```

Index 0 is UP (`x+0, y-1`), 1 is DOWN, etc. Matches the `Direction` enum exactly.

## The main function

```c
Direction ai_decide(const Game *g);
```

Returns the direction to move **right now**. Called every tick when in `MODE_AI`.

### Step 1 — initialize BFS

```c
for (i = 0; i < AI_MAX_H; i++)
    for (j = 0; j < AI_MAX_W; j++) {
        visited[i][j] = 0;
        parent[i][j] = -1;
    }

qhead = 0; qtail = 0;
qx[qtail] = hx;          // head position
qy[qtail] = hy;
qtail++;
visited[hy][hx] = 1;
```

Reset visited/parent. Push the snake's head into the queue.

### Step 2 — the BFS loop

```c
while (qhead < qtail) {
    cx = qx[qhead]; cy = qy[qhead]; qhead++;     // dequeue

    if (cx == fx && cy == fy) { found = 1; break; }   // reached food!

    for (d = 0; d < 4; d++) {
        nx = wrap_x(cx + DX[d], g->board_w);          // wraparound
        ny = wrap_y(cy + DY[d], g->board_h);

        if (nx < 1 || nx > g->board_w - 2) continue;  // outside playfield
        if (ny < 1 || ny > g->board_h - 2) continue;
        if (visited[ny][nx]) continue;                // already saw it
        if (!(nx == fx && ny == fy) &&
            is_blocked_by_snake(g, nx, ny)) continue; // body in the way

        visited[ny][nx] = 1;
        parent[ny][nx] = d;                           // we got here via direction d
        qx[qtail] = nx; qy[qtail] = ny;
        qtail++;                                      // enqueue
    }
}
```

Pop a cell, try its 4 neighbors. If a neighbor is in-bounds, unvisited, and not blocked by a snake, mark it and enqueue it. Stop when we pop the food cell.

### Step 3 — reconstruct the path

```c
trace_x = fx; trace_y = fy;
first_dir = -1;
while (!(trace_x == hx && trace_y == hy)) {
    int pd = parent[trace_y][trace_x];
    first_dir = pd;
    trace_x = wrap_x(trace_x - DX[pd], g->board_w);
    trace_y = wrap_y(trace_y - DY[pd], g->board_h);
}
```

Walk backwards from food to head. At each cell, `parent[y][x]` tells us which direction we came **from** (relative to the predecessor) — so the predecessor is `(x - DX[pd], y - DY[pd])`. By the time we reach the head, `first_dir` holds the **first** step of the path = the move to make.

### Diagram of reconstruction

```
parent[][]:        the direction USED to enter each cell.

  head H is at (3,2).  food F is at (4,0).

  ┌──┬──┬──┬──┬──┐
  │  │  │  │R │R │     R = entered from "RIGHT", i.e., from the left
  │  │  │D │R │  │     D = entered from "DOWN", i.e., from above
  │  │D │H │  │  │
  │  │  │  │  │  │
  │  │  │  │  │  │
  └──┴──┴──┴──┴──┘

walk back from F = (4,0):
   cell (4,0): parent = R  → first_dir = R, predecessor = (3,0)
   cell (3,0): parent = R  → first_dir = R, predecessor = (2,0)
   ... 
   cell (3,1): parent = U  → first_dir = U, predecessor = (3,2) = head ✓

So the FIRST step from head was UP (last loop iteration).
Return Direction::UP.
```

### Step 4 — fallback when no path exists

```c
if (dir_is_safe(g, g->p1.dir)) return g->p1.dir;
for (d = 0; d < 4; d++) {
    if (g->p1.length > 1 && d == opposite_dir(g->p1.dir)) continue;
    if (dir_is_safe(g, (Direction)d)) return (Direction)d;
}
return g->p1.dir;
```

Sometimes the snake gets boxed in by its own body and there's no path. We fall back to: prefer continuing forward; otherwise pick **any** safe direction; otherwise return the current direction (snake is doomed — let the death animation play).

### `dir_is_safe` helper

```c
static int dir_is_safe(const Game *g, Direction d) {
    int hx = g->p1.head->x, hy = g->p1.head->y;
    int nx = wrap_x(hx + DX[d], g->board_w);
    int ny = wrap_y(hy + DY[d], g->board_h);
    if (is_blocked_by_snake(g, nx, ny)) return 0;
    return 1;
}
```

"If I move in direction d, will I hit my own body next tick?"

## Performance

For a 200×60 board, BFS visits at most 12,000 cells per tick. With direction checks etc., that's roughly 50,000 operations. At even 50 ticks/sec the AI uses ~2.5 million ops/sec → microseconds of CPU. **Cheap.**

---

## Limitations (deliberate)

- **No look-ahead.** BFS only finds the path *now*. The snake might walk straight into a self-trap (eat the food, then be stuck). A more advanced AI would simulate ahead.
- **Greedy** — always go for the food, even if it means cornering yourself.
- **No Hamiltonian cycle.** The "perfect" snake AI follows a fixed cycle that visits every cell — guaranteed never to die. We chose simpler over perfect.

## 📢 Presentation Script — "AI"

> "The AI uses **breadth-first search** — the classic shortest-path algorithm — and runs it every single tick.
>
> We start by pushing the snake's head into a queue. Then we pop cells one by one, look at the four neighbors, and if a neighbor is in-bounds, unvisited, and not occupied by a snake body, we mark it visited and enqueue it. We also remember **which direction** we arrived from in a `parent[][]` table.
>
> When we eventually pop the food cell, we have the shortest path. We then walk backwards from the food to the head, using the `parent` table to step backwards — and the **last direction we saw** during that walk is the first step of the actual path. That's the move we return.
>
> If there's no path — say the snake has trapped itself — we fall back to any safe direction. The whole module is about 180 lines and runs in microseconds."
